что делает каждый блок в коде:

    1. Импорты и определение переменных:
В этом блоке происходит импорт необходимых модулей и определяете переменных, таких как FastAPI, Enum, choice и т. д.
Также определена переменная ID, которая используется для присваивания уникальных идентификаторов задачам, и список tasks, представляющий список задач.
    
    2. Создание перечисления для статуса задачи:
Здесь создается перечисление Status для определения статусов задачи - 'не выполнена' и 'выполнена'.

    3. Модели данных:
TaskIn - модель для входных данных при создании задачи, содержит заголовок, описание и статус задачи.
Task - модель задачи, наследуется от TaskIn, добавляет уникальный идентификатор для задачи.

    4. Генерация начальных задач:
Функция _generate_tasks() заполняет список задач tasks начальными значениями для демонстрации.
    
    5. Определение главной страницы:
Обработчик маршрута '/', который возвращает простую HTML-страницу.

    6. Получение списка всех задач:
Обработчик маршрута '/tasks', который возвращает список всех задач или генерирует новые задачи, если список пуст.

    7. Получение задачи по идентификатору:
Обработчик маршрута '/tasks/{task_id}', который возвращает задачу по указанному идентификатору.

    8. Создание новой задачи:
Обработчик маршрута '/tasks', который добавляет новую задачу в список задач.

    9. Обновление задачи по идентификатору:
Обработчик маршрута '/tasks/{task_id}', который обновляет информацию о задаче по указанному идентификатору.

    10. Удаление задачи по идентификатору:
Обработчик маршрута '/tasks/{task_id}', который удаляет задачу по указанному идентификатору.

    11. Запуск приложения:
Если файл запускается как основной, сервер FastAPI будет запущен на локальном хосте и порту 8000.
